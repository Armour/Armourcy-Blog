Ubuntu下翻墙的两种姿势 =w=

代码宅
  • Firefox
  • Goagent
  • Google Chrome
  • Ubuntu
  • ssh
  • vps
  • 翻墙

重装了一遍Ubuntu 并花了一上午弄各种环境配置 最后想起来妈蛋我还没翻墙！

发现想用 google chrome + goagent 翻墙的话必须先要翻墙下载一些东西才行，这TM不死循环了么。。

于是乎我弄了半天 先用 firefox + ssh 翻墙成功然后搞定了goagent，这样便掌握了2种不同的翻墙姿势！=w=

 

首先是 ssh + firefox 翻墙：
1. 打开Terminal 输入如下命令：
$ sudo ssh -qTfnN -p XXXX(ssh到你自己server上的的端口) -D 7070 username@servername
[sudo] password for **：（输入本机super user密码）
username@servername’s password:（输入SSH帐号密码）

2. 设置firefox浏览器
编辑-》首选项，然后-》高级-》网络-》设置，选择“手动配置代理“。
SOCKS主机：localhost，端口：7070。其他的都不用改。然后点确定。

3. 设置域名解析
打开firefox
在地址栏输入：about:config 然后回车
设为true

这样就OK了～

 

 

接下来是google chrome + goagent的翻墙：

先是下载一些必要的东西，包括google-appengine-xxx.zip以及goagent的最新版。

下载的网址如下：

1> 如果你能翻墙的话（比如像之前的firefox）：

https://cloud.google.com/appengine/downloads?hl=zh-CN

选择 Google App Engine SDK for Python 里的zip下载

https://github.com/goagent/goagent

选择最新版goagent下载

2>如果你不能翻墙的话，我这里有提供比较新版本的=w=：

Google Appengine 1.9.15.zip

Goagent3.2.2

假设以上两个文件都下载到了 “~/下载” 目录里（即 /home/用户名/下载），那么终端执行以下命令：

$ cd /home/用户名/下载；下面的步骤就是将刚才的两个文件安装到此目录中；

$ unzip google_appengine_1.9.15.zip，解压缩

$ upzip goagent-goagent-3.2.2.zip google_appengine/

接下来是配置和上传数据:

在goagent文件夹内找到proxy.ini，修改［gae］下的appid为你自己的google账户，保存退出

然后我们运行goagent：

$ sudo python {your goagent path}/local/proxy.py

同时另开一个terminal开始上传：

$ sudo python {your goagent path}/server/uploader.zip

上传的过程中提示输入邮箱（注册 appengine 使用的 gmail）以及 gmail 的登陆密码；如果 gmail 设置了应用专有密码，第一遍输入完登陆密码并回车后，会提示再输入一遍，这一遍就是输入应用专有密码；如果没有设置应用专有密码就不需要第二次输入了。应 用专有密码就相当于一个密保问题一样的东西。

喝杯茶等一会儿应该就弄好啦，最后大概结果如下：

最后是浏览器代理设置:

Chrome: 安装 SwitchySharp 插件，打开 SwichySharp 设置–导入/导出–从文件恢复，然后导入goagent文件夹下的SwitchyOptions.bck
导入证书(不然可能不能正常打开https网页):chrome 设置–>高级–>证书管理–>选择第三个选项卡–>选择 goagent/local/ca.crt， 保存

（FireFox: 安装 FoxyProxy 插件，打开FireFox->选项->高级->加密->查看证书->证书机构->导入证书， 选择 goagent/local/ca.crt, 勾选所有项，导入，保存）
