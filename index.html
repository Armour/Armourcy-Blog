Ubuntu下翻墙的两种姿势 =w=

重装了一遍Ubuntu 并花了一上午弄各种环境配置 最后想起来妈蛋我还没翻墙！

发现想用 google chrome + goagent 翻墙的话必须先要翻墙下载一些东西才行，这TM不死循环了么。。

于是乎我弄了半天 先用 firefox + ssh 翻墙成功然后搞定了goagent，这样便掌握了2种不同的翻墙姿势！=w=

 

首先是 ssh + firefox 翻墙：
1. 打开Terminal 输入如下命令：
$ sudo ssh -qTfnN -p XXXX(ssh到你自己server上的的端口) -D 7070 username@servername
[sudo] password for **：（输入本机super user密码）
username@servername’s password:（输入SSH帐号密码）

2. 设置firefox浏览器
编辑-》首选项，然后-》高级-》网络-》设置，选择“手动配置代理“。
SOCKS主机：localhost，端口：7070。其他的都不用改。然后点确定。

3. 设置域名解析
打开firefox
在地址栏输入：about:config 然后回车
设为true

这样就OK了～

 

 

接下来是google chrome + goagent的翻墙：

先是下载一些必要的东西，包括google-appengine-xxx.zip以及goagent的最新版。

下载的网址如下：

1> 如果你能翻墙的话（比如像之前的firefox）：

https://cloud.google.com/appengine/downloads?hl=zh-CN

选择 Google App Engine SDK for Python 里的zip下载

https://github.com/goagent/goagent

选择最新版goagent下载

2>如果你不能翻墙的话，我这里有提供比较新版本的=w=：

Google Appengine 1.9.15.zip

Goagent3.2.2

假设以上两个文件都下载到了 “~/下载” 目录里（即 /home/用户名/下载），那么终端执行以下命令：

$ cd /home/用户名/下载；下面的步骤就是将刚才的两个文件安装到此目录中；

$ unzip google_appengine_1.9.15.zip，解压缩

$ upzip goagent-goagent-3.2.2.zip google_appengine/

接下来是配置和上传数据:

在goagent文件夹内找到proxy.ini，修改［gae］下的appid为你自己的google账户，保存退出

然后我们运行goagent：

$ sudo python {your goagent path}/local/proxy.py

同时另开一个terminal开始上传：

$ sudo python {your goagent path}/server/uploader.zip

上传的过程中提示输入邮箱（注册 appengine 使用的 gmail）以及 gmail 的登陆密码；如果 gmail 设置了应用专有密码，第一遍输入完登陆密码并回车后，会提示再输入一遍，这一遍就是输入应用专有密码；如果没有设置应用专有密码就不需要第二次输入了。应 用专有密码就相当于一个密保问题一样的东西。

喝杯茶等一会儿应该就弄好啦，最后大概结果如下：

最后是浏览器代理设置:

Chrome: 安装 SwitchySharp 插件，打开 SwichySharp 设置–导入/导出–从文件恢复，然后导入goagent文件夹下的SwitchyOptions.bck
导入证书(不然可能不能正常打开https网页):chrome 设置–>高级–>证书管理–>选择第三个选项卡–>选择 goagent/local/ca.crt， 保存

（FireFox: 安装 FoxyProxy 插件，打开FireFox->选项->高级->加密->查看证书->证书机构->导入证书， 选择 goagent/local/ca.crt, 勾选所有项，导入，保存）

OpenCV2.4.9 + Win7 + VS2013 环境配置

因为要准备中控杯 所以就着手学习着试试玩OpenCV

环境配置的话linux下还是比较好弄的 windows下简直恶心 遇到了各种问题 特此记录一下完整版的过程 以备查询

首先明确一下我们的配置：

Win 7 (为了玩游戏还是没忍心删掉 = =)

OpenCV 2.4.9 (记得配置环境变量, 在win7环境变量的PATH里面添加opencv的lib和bin两个文件夹路径)

Visual Studio 2013 Ultimate

为了锻炼英语能力(其实是为了看起来逼格更高), 我决定用英文来记录过程, 那么我们开始吧~

 

 

1.Using VS2013 create a new project which is：

    1) .NET Framework 4.5

2) Win32 Console Application

3) empty project</pre>

 

 

 

2.Add “x64” in solution platforms

First we choose Configuration Manager to edit it

Add a “x64” platform if not found

This is what we want : choose and use the “x64” platform

 

 

 

 

3. Add your “Debug|x64” Property Sheet.

Choose “VIEW->Property Manager” and find “Debug|x64”, then right-click it and choose “Add New Project Property Sheet”；


name it whatever, here we assume it “OpenCV_Debug.props”;
then double-click “OpenCV_Debug and it will pop up a property page;

4. Choose “VC++ Directories”, change follow path:

Include Directories => “{replace this with your opencv install path}\build\include”

Library Directories => “{replace this with your opencv install path}\build\x64\vc12\lib”

(all allow inherit from parent or project details)

Apply it.

 

 

 

 

5. Choose “Linker”, change “Input”->”Additonal Dependencies” to follow:

opencv_calib3d249d.lib

opencv_contrib249d.lib

opencv_core249d.lib

opencv_features2d249d.lib

opencv_flann249d.lib

opencv_gpu249d.lib

opencv_highgui249d.lib

opencv_imgproc249d.lib

opencv_legacy249d.lib

opencv_ml249d.lib

opencv_nonfree249d.lib

opencv_objdetect249d.lib

opencv_ocl249d.lib

opencv_photo249d.lib

opencv_stitching249d.lib

opencv_superres249d.lib

opencv_ts249d.lib

opencv_video249d.lib

opencv_videostab249d.lib
Apply and Save it.

 

 

6. To avoid fatal error LNK 1112: 模块计算机类型“X86”与目标计算机类型“x64”冲突:

In solution manager, we right-click “Debug|x64” and choose “Properties”

then choose “Linker”->”Advanced”->”Target Machine” , change it to X64~

//You can also skip this one to see if this error comes or not :D

 

 

 

7. To avoid the error “……, Can’t find or open the PDB file” :

Choose “DEBUG” -> “Options and Settings”->”Debugging”->”General”, allow the “Enable source server support” option.

then change to “Symbols” and enable the “Microsoft Symbol Servers”, save , quit and you can run your program.

(After this step, you program will load many .dll from Microsoft Symbol Servers, so it will be vary slow first time)

(So when first time run over, we can disable the “Microsoft Symbol Servers”, then it will run well and quickly next time! )

 

 

8.All finish！ Here is a simple .cpp program you can start with :)

#include <opencv2\opencv.hpp>

int main(void) {

IplImage* img = cvLoadImage("Test.jpg");

cvNamedWindow("Example1", CV_WINDOW_AUTOSIZE);

cvShowImage("Example1", img);

cvWaitKey(0);

cvReleaseImage(&amp;img);

cvDestroyWindow("Example1");

}


if everything works well, you will get result like this: (depends on your “Test.jpg”, so don’t forget to add it )

 

 

大概就这样啦~

PS： 如果遇到一些提示问题说找不到opencv_XXX.dll 那么你就从opencv的x64的dll库里面找到复制到system32文件夹里面（当然也可能是SysWow64文件夹, 根据错误信息来判断）然后命令行运行“regver32 /s opencv_XXX.dll” 即可.

在vps上建立水水的OJ~

最近zy想弄一个oj给内部使用,于是就着手学习着搭建了一个简易oj(采用hust oj模板).
首先ssh到ubuntu server~

ssh -p XXXX(偷偷的改掉了ssh的port端口) root@104.131.134.92
然后安装LAMP！(当然LAMP环境搭建完成后就可以很方便的使用各种[**wordpress**](https://www.digitalocean.com/community/tutorials/how-to-install-wordpress-on-ubuntu-12-04)模板搭建属于自己的blog了,这里就不详细介绍啦~ 1.先是Apache~ sudo apt-get update sudo apt-get install apache2 2.再是安装Mysql~ sudo apt-get install mysql-server php5-mysql sudo mysql_install_db sudo mysql_secure_installation 3.最后是安装PHP~ sudo apt-get install php5 libapache2-mod-php5 php5-mcrypt 
  此处要修改dir.conf文件使index.php优先级高于index.html：
sudo vim /etc/apache2/mods-enabled/dir.conf 
  将index.php提至最前,修改完如下：
<IfModule mod_dir.c> 
    DirectoryIndex index.php index.html index.cgi index.pl index.xhtml index.htm

</IfModule>

  蓝后可以重启apache2了~

sudo service apache2 restart
配置环境完成后就可以安装hust oj模板了,可以通过github获取打包的源码,然后cd到hust oj目录,运行:
sudo bash install-interactive.sh
静静等待一段时间后就安装好了！~
之后的细节配置还有创建数据库啥的都可以参照hustoj自带的README！;
